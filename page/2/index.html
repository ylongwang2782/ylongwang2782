起因

我开始了解电影的各种技术，从画面上，从声音上，无一不让我惊讶。跑遍了几乎所有特效听，数字IMAX,两家dolby影院，cinity影厅，4dx影院。顶级的还差激光imax没有看。

前置计划

hao4k

了解家里的开关结构，智能开关面板更换老式开关超详细操作

发表于 2021-02-24 更新于 2022-03-28 Waline： 阅读次数： 本文字数： 2k 阅读时长 ≈ 2 分钟

这篇文章，记录一下更换开关所需要的一些简单电学知识和走进开关内部结构。

以及自己的一些实际更换操作

开关的内部结构

要进行开关的更换，特别是用智能开关和旧机械开关构造完全不一，所以了解开关的内部结构，了解电路的基础结构尤为重要。

废话不多说，直接上图。由我实际的总结，可能不全，但包括了最难的三开双控解析，应付大部分的情况都没问题。

双开单控

首先拿到一个老式开关面板 ，进行一步步拆解

现在拆解的，就是最简单的双开单控开关。

（所谓的几开几控，几开指这个开关一共有几个开关~~

几控，单控，即单个灯只有一个开关控制，而双控则是单个灯由两个开关共同控制，两个开关同时控制灯的打开和关闭。）

下面开始仔细分析开关的结构，它到底是怎么执行开关这个功能的呢？

由这个简单的解析可以看到，火线由下面的柱头进入，通过铜传导至两个弹片，由弹片控制与上面两个柱头的接触。上面两个柱头则分别接两个灯泡。

弹片开闭两种状态如下

开关合上，灯亮

开关与上部铜柱接触，则回路导通，灯亮

弹片翘起来，即回路断开，灯不亮。

双开双控

先了解一下双控的基本连线

可以看到，只要是双控开光，则必须由三个柱头构成。中间由两根火线L1L2连接，由单根火线接入接出。

双控开关AB，必须同时连接L1或者L2才能使得回路导通，让灯通电。

所以不难解释生活中的情况。

当双控灯亮着的时候，不论将开关A还是开关B拨动，都会使得回路断开然后使得灯泡断电。

当双控灯灭着的时候，不论将开关A还是开关B拨动，都会使得回路变为导通。

（这其实就是与或非中的与，参考最简单的01真值表就可以得到结果）

继续拆解一个双控开关

一个双开双控开关的正反两面。

由背面的标识和内部的结构对应不难得出以下结论：

  1. L1/L11/L12和L2/L21/L22分别为两套独立的双控开关。三个柱头为一组。和上面的电路图刚好对应。

  2. L1和L2分别是两组中的火线进入或者火线出柱头。

  3. L11/L12和L21/L22则是两个中间柱头

    对应电路图如下

另，如若将双控开光用于单控用，也是可以的。只需将一个铜柱不接就可以了。

三开双控

三开双控的开关应该是最难的开关了。但实际上内部结构也十分简单。

一共9个开关，每三个一组，不难看出三开双控开关和双开双控开关书本同源。

三开双控开关，是最复杂的开关，但也是适用面最广的开关，它基本适用于家庭的任何场景，所以在实际操作的时候一定要注意它的变形。

实例操作图解

实际操作中，注意安全，断电和电笔测电很重要。

断电保护基本安全。电笔则可以通过排除法，了解电路结构。实际生活中，因为电工的技术水准参差不齐，家庭电路的实际安装有可能令人瞠目结舌，并不一定就是书本上最简单的。所以了解实际结构很重要。

用我实际更换家里的一套：三开双控开关为例

家里的楼梯灯共有三个，一楼负一楼负二楼各一个。

使用情况则是，负一楼是三开双控开关，一楼负一楼都是单开双控开关。所以，负一楼的可以控制三层楼梯的开关，而一楼负二楼的则只能控制各自楼层的灯泡。

实际连线图

负一楼的三开双控开关，接线的复杂程度，令人头昏眼花。

一楼的双控单开开关

负二楼的双控单开开关

看了三个开关，我第一时间就有所疑问，为什么负二楼的开关是被短接了的？和理论上的双控连线图出了差误。

而后通过电笔的仔细测量和排除，最后得出结论

其中，绿色的六根铜柱为负二楼的灯

黄色的六根铜柱是一楼的铜柱

蓝色圈则是负一楼的单控灯，嘿嘿

因为实际操作中包括了短接，串接，空接的情况，所以会导致对于模型的建立产生极大的困难。不过只要电笔玩得好，猜想加证明，不难得出最后模型。

难点：1.一楼灯和负一楼灯都是由一根火线供电，一楼的电更是由火线串接而来。

2.负二楼的火线并不是由负一楼的开关提供！而是由负二楼接入，再由负一楼接出。这点真的把我难住了很久。果然实际和理想差别很大┭┮﹏┭┮

智能开关更换

智能开关共有两种，单火线版本，和零火线版本，这点网上大佬们就解释得很清楚啦。

而我这一个

则是一个单火线的版本。

值得注意的是：

  1. 智能开关只需要接入一个火线，然后其他柱头依次接灯泡即可
  2. 智能开关不能适用于机械式的双控开关，要实现多控则需要配合遥控开关使用。虽然不能实现机械式双控，但智能开关可以更加简单的实现N控，这点是传统开关做不到的。
  3. 单火版的智能开关因其特殊性，必须串接一个电容在灯具两端，这是一个难点
  4. 新一代的房屋大多是零火版，零火版的智能开关不仅安装简单，而且价格上也更加实惠。

网络环境已经写完了。

下面写一下自己捣鼓软路由的工程。

虽然当初懵懵懂懂的我，甚至还没有一丁点的网络基础知识，就直接接触到了软路由，一脸懵逼的照搬网上的教程，把这个软路由给搞定了，但其中，当真是吃了不少亏，纠结了一个又一个下午与晚上。

现在当初的那套系统，基本都被我拆的拆，换的换，这么多坏与修之间，再无数次放弃与挣扎中，我在一年内，大概是终于，最后，完成了这套系统。

那一个个慢慢摸索的夜晚，哎，当真是一言难尽。机器是死的，它出任何问题，它不会说话，不会告诉你它哪里不舒服，哪里不行了，哪里坏了。只会突然间，就开不了机了！突然间，就未相应了！突然间，按照原来的教程已经继续不了了，只能换一个教程，从头再来。还有可能，一次重启，可能就是永别。重启完后，你这套系统就再也运作不了了。真的惨淡。

吐槽就到这。下面开始正文。

1.all in one时期

初入门道的我，作为垃圾佬的本质，我决定采用all in one，这样就可以用极少的成本就可以拥有更多的功能。

于是我按照网上大佬们的教程，这里引用范大佬的文章

什么值得买上的文章

最初的我，按照大佬的文章，购买了j1900双网口蜗牛星际，按照文章的一步步教程。

在蜗牛上安装了pve虚拟机系统，并在虚拟机系统上面，建立了lede（openwrt的一个版本）和nas的二合一系统。

不过因为实践和理论经验都较差，我在这个时期经常因为系统出错，而需要将机器进行多次的刷机和软件维修。并因为它同时也是家里的主路由器，这就导致了系统每一次的瘫痪，不管是nas坏了，还是软路由坏了，我都必须要把整个系统一起拆下来修。这就导致了家里经常断网！

经过长达十几天的尝试，最终这个系统后来还是正常运行了大约3个月。

这期间，通过数次的错误，我总结如下

  1. 软路由的wan口和lan口一定要分清楚
  2. 只有软路由开启dhcp功能，并且其他作为ap的路由固定ip地址一定不能与软路由网关的ip地址冲突。
  3. 黑群晖系统会时而自动关机。并非出错。
  4. 蜗牛的关机键无法正常使用，通过pve软系统建立的lede和nas系统不能通过外部关机键强制关机，而直接拔电源则会带来很大的硬盘损毁可能性。
  5. openwrt系统会时常断网，这时只需要重启wan口就可以了。

最终，这个all in one系统因为我老爸的强制关机，导致硬盘的坏损而宣告结束。

总结，多合一系统，对于折腾着玩的朋友完全可以，而要当日常主力机使用，要融入生活中，all in one系统常常因为其多系统性，而导致多故障，常常导致一坏全坏的惨淡局面。

解决大户型的网络问题，附实际操作图

发表于 2021-02-22 更新于 2022-03-28 Waline： 阅读次数： 本文字数： 1.5k 阅读时长 ≈ 1 分钟

本文旨在总结自己经历和经验，文笔较差，请见谅。

在下家里原来的情况是最普遍的一个情况，一个光猫，一个路由器，全家上下都靠这个路由器来上网。但麻烦就麻烦在，因为家里的是洋房，除了一楼外，还有负一楼和负二楼。这就导致了如果只在一楼安装一个路由器，下面两层楼基本就没有信号。

于是为了解决这个问题，我在网上苦苦寻觅，终于找到结果。

理论构想

先理解两个名词

解调：光信号到电信号的过程

拨号：由光信号转为电信号的过程

路由：分配ip的过程

光纤入户后，必须经过解调、拨号和路由功能才能被家庭使用。常规家庭中，由光猫进行解调、拨号和路由。而在早期，拨号通常由电脑自己拨号上网，所以当时的网民，打开电脑后，必须先进行网络连接，拨号成功后才能进行上网。而现在，因为光猫集合了拨号和路由功能，于是下面的设备只需要插上网线或者连上WiFi就可以直接上网。

1.1由于光猫本身也可以发送WiFi信号，所以在有些小户型，则可以完全不需要安装额外的路由器来扩展WiFi信号就可以实现全屋覆盖。这就是单光猫情况。

1.2而常规的家庭则通常接入一个路由器连接到客厅，进行WiFi范围拓展。这时由光猫解调，路由器拨号和路由，，这就是单光猫+单路由器情况。

因此，要实现三层楼的WiFi覆盖，就很简单了。只需要，多加几个路由器！

1.3刚好家里剩下了几个淘汰的路由器，于是就每层楼安装一个子路由器，以此拓展WiFi覆盖范围，而这种拓展WiFi范围，WiFi信号发射器就叫做无线AP（Access Point）。

路由器如果只插lan口，则只实现ap功能。当然也可以买专业的ap，不过能废物利用怎么能不用上呢。实现单光猫+多路由。

1.4但往往因为光猫性能较差，而且要实现无缝漫游。于是就必须使用另一个路由器来进行拨号和路由。而这个路由器就是AC无线控制器。实现单光猫+主路由器+副路由器。

主路由器的选择，可以随便选择一个路由器，可以是成品的硬路由，也可以是更高性能的软路由。我选择的是openwrt的软路由系统进行路由和更多神奇功能。

这样，一个完整的局域网就构造好了。

另，实际上因为网线和网口的限制，通常需要将从主路由器出来的一根网线通过交换机，变成多根网线输出到每一个子路由器上

于是目前的机构是：

光猫（解调）——主路由器AC（拨号、路由分配局域网ip地址）——交换机——多个子路由器AP（扩展WiFi信号）——各种用网设备（手机、电脑等）

这样就可以实现全屋WiFi无缝漫游啦。

认识局域网

发表于 2021-02-22 更新于 2022-03-28 Waline： 阅读次数： 本文字数： 874 阅读时长 ≈ 1 分钟

局域网，这个早在几十年前就创造出来的成品，随着互联网的普及，局域网似乎并没有被人们所认识和密切接触。

人们大部分则是直接与互联网接触。无论是电脑的联网，手机的联网，这里的“网”几乎都是特指互联网，即与外面的网络相连接，这样使用网络的人才能与外部交互，查百度，刷b站，全部都是建立在互联网的基础上。

而今天要说的则是互联网的前辈：局域网。局域网，顾名思义是局部上的网路，它虽然不为人知，但却深入人们的生活之中。局域网不需要外部网络，只需要一台只能设备即可。现在随处可见的路由器就是局域网的一个体现。路由器将接收到的网络信号通过编码，通过有线或者无线的方式将网络信号传输给另一端的用户。

路由器产生WiFi，手机连接WiFi继而连接到互联网，俗称“有网”。而在这其中，局域网就悄悄的产生其作用。路由器作为一个终端，发挥它的路由功能，给每一个“设备”都分配一个ip地址。注意，这是局域意义上的ip地址，只有每一个设备被分配到ip地址后，其才能从路由器那里接受到网络，从而连接到互联网，可以说路由器是一个中转站，手机与互联网之间的中转站。

而作为分配ip地址，发送网络的“老大”，路由器自然成了这个局域网的核心。所有常见的局域网均由这个路由器分配产生。就像开宗立派的开山鼻祖，所有这个“局域网”宗派内的“弟子”都由这个路由器产生和管理。

一个分配ip地址，发送接收网络信息的路由器，就是一个最简单的局域网。就算是光杆司令，这个路由器只插了一个电源，没有插其他网线，没有任何其他设备连接，它也是自成一体，称为一个独立的局域网。当然这样一个最简单的局域网是没有连接互联网功能的，也就是说，它“没网”。

而更常见的，在每个家庭几乎都有的局域网情况是，一个“光猫”连接到路由器上，然后所有需要网络的设备全部连接到路由器上。这样就“有网”可用了。

另一个现在越来越常见的东西，热点。其实也是一个小型局域网，不论是手机热点还是电脑热点，和路由器基本都是大同小异，都是其他设备连接到这个终端上，并且“有网”。执行热点功能的手机\电脑其实就是一个路由器。他们同时执行着一个功能“路由”。
