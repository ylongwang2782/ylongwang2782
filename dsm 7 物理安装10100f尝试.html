DSM_7物理安装10100f尝试

发表于 2022-08-13 更新于 2023-03-03 Waline： 阅读次数： 本文字数： 3.2k 阅读时长 ≈ 3 分钟

前言

由蜗牛换到10100f新设备，真的太方便了！长痛不如短痛！

看看我的成品图

更换原因

19年整的一个蜗牛星际C双的设备，我实在是忍受不了这种缓慢的速度了

兴致来了，直接开始利用家里闲置的十代u开始整黑裙

硬件背景

之前淘汰下来看电视用的tvpc，配置如下：

10100f，h410m爆破弹，8G金士顿2400，网吧拆机长城400w电源（是不是有危险，哈哈啊哈

另外还有一个1030独显

引导文件

一个是synoboot.img 类的引导文件，该文件需要找特殊的

一个是.pat 的固件，引导文件对应的版本在官网下载

（我一直以为引导文件是绑定U的，原来是绑定一个DSM版本;>

刷机步骤

记录u盘参数

在计算机管理-设备管理器-通用串行总线设备找到如下图

USB\VID_1F75&PID_0918

改引导文件需要使用

将引导文件中vid pid ox后四位对应修改

最终文件

引导：DS3622xs_7.x.img

引导下载地址：GXNAS网盘

固件：DSM_DS3622xs+_42661.pat

换了很多个引导只有这个引导成功了，其他都检测不到ip，也不知道什么原因

固件在官网直接下载3622xs最新的固件能够使用。

引导文件修改

下载好的引导文件，需要使用OSFMount挂载虚拟镜像，在镜像中修改grub.cfg文件，初始刷机主要是修改vid和piv两个数值。

其余的参数可以保持默认

值得注意的是DSM7必须在sata1口上插上一个硬盘。

刷入u盘

将修改完成后的整个img引导文件用win32diskmgr刷入删除分区格式化（diskguines）后的u盘中。

在使用win32disk刷u盘的时候，会遇上该问题，需要手动解决

Win32DiskImager写img到u盘报“拒绝访问”解决办法_NorthrendSnow的博客-CSDN博客_win32diskimager 拒绝访问

安装

将u盘插上机器上开机

pc访问群晖，进入安装界面，按照一般步骤来就可以了。

硬盘参数

使用正常群晖的安装步骤，安装成功后，需要关闭群晖，重新设置引导u盘中参数（因为之前设置的只有一个硬盘，现在我有四个硬盘）

引用别人的介绍

#在群辉系统中，一个控制器默认只能控制9个硬盘，所以SataPortMap=的数值范围只能是0-9，0则表示屏蔽该控制器，不可以设为SataPortMap=0来隐藏引导盘！

#如果有多个控制器的（每个主板具体有几个控制器，请查阅主板说明书），则SataPortMap=后面的数值有几位就表示有几个控制器。比如SataPortMap=148，表明第一个控制器有1个硬盘，第二个控制器有4个硬盘，第三个控制器有8个硬盘。（包括sata和msata，但不包括m2）

#DiskIdxMap=这个参数值，按顺序从左到右每两位数为一个控制器的盘序数值，这两位数为16进制数，如果数值小于等于9或者大于10的，需要在数值前面加个0。比如DiskIdxMap=000105的意思是，第一个控制器的磁盘（引导盘）从序号1（00为序列1）开始往后排，第二个控制器的4个硬盘从序号2（01为序号2）开始往后排，第三个驱动器的8个硬盘从序号5（05为序号6）开始往后排。此时在群晖DSM7.X的系统里面看到磁盘序号就会从磁盘1（引导盘）到磁盘13。

#这里有个小技巧，可以将控制器的盘序设置超过当前最大磁盘数量。例如我想把第1个控制器的引导盘隐藏，可以改成DiskIdxMap=0C0004，引导盘在群晖系统里面就看不到了，此时在群晖DSM7.X的系统里面看到磁盘序号就会从磁盘1到磁盘12（引导盘被隐藏了），同时还治愈了很多患有强迫症的童鞋！

#SasIdxMap为SAS硬盘的个数，默认使用SATA硬盘的时候设置为0。

我的理解是：

关于SataPortMap

星际蜗牛有两个msata，四个sata口，有两个控制组，所以蜗牛星际的SataPortMap=24

我的h410m爆破弹只有四个sata口，所以控制组只有一个，所以SataPortMap=4

关于DiskIdxMap

h410m只有一个控制器，所以直接从00开始就ok了。

所以最终h410参数为

SataPortMap=4

DiskIdxMap=00

其他主流的pc主板应该是差不多的，除了有的主板有6个sata

系统迁移

迁移硬盘

调整硬盘参数为4后，将原来蜗牛上的硬盘全部重新插拔到新机子上

启动群晖，在存储管理器中，在线重组，点击确认就完成硬盘迁移了

真的不要太简单，比我想象轻松太多了

半洗白

为什么要半洗白

因为photo要查看heic，必须要认证用户安装AME（安装和授权 Advanced Media Extensions）

安装步骤

  1. 在这里 https://archive.synology.com/download/Os/DSM/7.1-42661 下载对应系统的虚拟机 Pat
  2. 参考该教程在群晖里安装虚拟机群晖中用VMM（Virtual Machine Manager）再虚拟安装一台群晖保姆教程_NAS存储_什么值得买 (smzdm.com)
  3. 在群晖助手中搜索到新启动的dsm，其中虚拟机的序列号应该就是我们要的sn了，图中的序列号为VA4WHJFV69HPZ


  1. 将群晖和虚拟机关机，然后更改u盘中引导文件的sn码为上述sn码

总结

  1. 时间花费：总体比较顺利，从开始行动，到调研相关知识，到开始动手操作，到最后的成功转移，全过程大约四个小时。
  2. 性能提升巨大。鸟枪换大炮，现在终于可以在群晖里为所欲为了，经过一天的测试，cpu占用率没有超过10%。后期可以考虑更换一个低压板U。
  3. 群晖系统迁移相当方便。从老系统到新系统，除了套件的内容不好转移，其余的内容几乎是直接全部搬迁。并且不论是什么U都可以使用（反正我用户端解码）有点Windows这种感觉了，以后想要搬系统的时候，直接找个板U刷上，插上硬盘就完事。
  4. 网络提升。之前的虚拟机蜗牛，最多只有400M，更换后直接变成1000M。
  5. 功耗。暂时没有考虑，估计最多比蜗牛多40W，一个月多十几块的电费。
  6. 迁移需要准备的事情：套件appdata全部备份，docker共享文件夹备份。

  2. 关于硬盘，目前的打算还是4X4T，不够用了再换
  3. 发掘更多的功能，让10100f跑起来
