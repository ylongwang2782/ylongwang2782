小米路由器mini刷Pandora

发表于 2022-04-06 分类于 校园网折腾 Waline： 阅读次数： 本文字数： 1.4k 阅读时长 ≈ 1 分钟

参考的视频为

小米路由器Mini刷Openwrt教程_哔哩哔哩_bilibili

然后以下的内容是我自己的总结

准备工作

一共要提前准备五个文件

mini路由器的开发版固件

http://www1.miwifi.com/miwifi_download.html

在支持网站ROM目录下载对应路由器的开发版固件

小米路由器mini的root密码和miwifi的ssh插件

https://d.miwifi.com/rom/ssh

登录小米的ssh网站，获取账号绑定的路由器root密码

以及对应路由器的ssh估计

breed

Index of / (hackpascal.net)

在支持网站中下载对应的路由器breed

PandoraBox

Index of /PandoraBox/Xiaomi-Mini-R1CM/stable/ (openwrt.org.cn)

在openwrt官方支持网站上下载即可

第一步：刷开发版固件

在路由设置中-系统状态-系统升级-手动升级

选择miwifi_r1cm_firmware_2e9b9_2.21.109

点击升级即可

升级过程期间一直是橙灯

大概几分钟后，指示灯由橙灯转为蓝灯后，刷新管理界面，即可进入设置界面

第二步：上传ssh功能

准备一个u盘，选择FAT32，然后格式化

将miwifi_ssh.bin文件拷入到格式化后的u盘根目录中

关键步骤

  1. 断电
  2. 然后按住reset键的不要松，同时给路由器上电（插上电源）
  3. 大概7秒左右，路由器开始闪橙灯
  4. 然后

断电，先按住reset键，然后上电，等待十秒左右，路由器开始闪红灯

第三步：使用ssh工具连接路由器

我使用软件的是putty

  1. 打开putty
  2. 输入小米路由器mini的默认网关地址192.168.31.1
  3. 用户名输入：root
  4. 密码输入：之前在网站中拷贝的root密码
  5. 敲击回车看到登录画面则算成功

第四步：上传breed固件

使用的软件是winscp

上传

  1. 打开winscp，文件协议选择SCP

  2. 依次输入登录地址：192.168.31.1 用户名：root，密码：xxxxx

  3. 打开/tmp地址

  4. 将breed-mt7620-xiaomi-mini.bin文件名更改为breed.bin（不更改也行，后续代码相应更改即可）

  5. 选择上一步打开的putty软件，输入mtd -r write /tmp/breed.bin Bootloader，敲击回车,看到如图所示代码

烧录

  1. 将U盘拔出
  2. 路由器断电
  3. 按住reset键
  4. 路由器上电
  5. 此时路由器一直闪烁蓝灯，可松开reset键
  6. 然后可以通过输入breed管理地址192.168.1.1，访问breed界面![image-20220405171158749](https://pic.imgdb.cn/item/624d55b7239250f7c5a296ed.jpg)

第五步：通过breed烧录pandora系统

  1. 登录breed管理界面
  2. 在固件备份中备份EEPROM和编程器固件（可选）
  3. 在固件更新中勾选固件，并选择之前准备好的pandora固件PandoraBox-ralink-mt7620-xiaomi-mini-squashfs-sysupgrade-r1024-20150608.bin
  4. 点击上传，跳转到确认界面
  5. 点击更新
  6. 等待几分钟更新完成后，重启电脑的网卡，以获得重新分配的ip地址
  7. 输入192.168.1.1即可到登录界面

使用python自动认证哆点

需要的软件为wireshark、putty、winscp

登录信息抓包

1.wireshark

  1. 将网线连接电脑
  2. 打开客户端，输入账号和密码
  3. 打开wireshark，点击以太网，点击start
  4. 登录哆点
  5. 大概十几秒后，点击左上角红色按钮停止
  6. File-Save As 保存为dr.pcapng

2.生成配置文件

  1. 登录在线配置生成器https://drcoms.github.io/drcom-generic/
  2. 选择对应的哆点版本
  3. 将之前获得的dr.pcapng上传
  4. 获得新的配置文件，点击SAVE保存配置文件drcom.conf
  5. 用记事本打开配置文件drcom.conf，再对应的password处填入密码并保存

在PC上测试（可选）

配置PC端python2的ide环境

我使用的是pycharm

配置PC端的python目的是为了方便测速

下载

  1. 登录到python官网Python Release Python 2.7.18 | Python.org
  2. 选择Windows x86-64 MSI installer下载
  3. 使用默认安装
  4. 在PyCharm: the Python IDE for Professional Developers by JetBrains下载pycharm

配置

  1. 在pycharm中file-setting-python interpreter
  2. 点击齿轮，选择Existing Environment 选择python2的exe文件

获得学校对应的py程序

drcom-generic/custom at master · drcoms/drcom-generic (github.com)

  1. 在GitHub项目上查找自己对应的学校并下载，我是CQU-huxi
  2. 下载后pycharm添加项目打开py文件
  3. 将之前生成的配置文件drcom.conf填入# CONFIG与# CONFIG_END
  4. 将哆点注销并推出，直接run查看是否成功联网

pandora安装python2

  1. download zlib_1.2.8-1_ralink.ipk and python-mini_2.7.3-2_ramips_24kec.ipk
  2. 将以上两个文件放置到/tmp目录下
  3. putty中cd /tmp然后opkg install zlib_1.2.8-1_ralink.ipk短暂时间后，将提示两个ipk都安装成功

以上的使用情况是我的小米路由器mini的安装情况，其余设备可能会有所不同

pandora运行drcom.py

  2. 在putty中运行drcom

卫星地面站系统资源能力评估方法研究

评估指标体系

空军工程大学的王远振等人

从对卫星服务的满足程度、地面站设备的利用率、重要卫 星弧段的服务保障度、系统整体生存能力和方案的鲁棒性五个方面

建立了卫星地面站设备配置方案的*评估指标体系***［5]。

评估指标

这些指标包括服务总时间比、平均任务调度成功率、接受设 备利用率、测控设备利用率、综合设备利用率、设备的生存能力、设备的可靠性、设备的 冗余度、设备硬杀伤概率、设备抗毁能力、设备软杀伤概率、设备抗干扰能力、设备机动 能力、系统增加卫星后的方案效能、某卫星变轨后的方案效能和紧急情况下的方案效能。

国防科技大学的金光等人

在深入研究地面站资源优化配置的基础上，分别从卫星用户需求和地面站资源拥有方需求两个方面建立评估指标体系，

评估指标

他们提出的评估指标包括：平均每天服务时间、平均每天服务圈数、平均每天满足要求率、 平均每圈满足要求率、 地面站设备利用率、 地面站设备总费用、 地面站服务 总时间和地面站系统可用度。

西安卫星测控中心的唐枚、李济生

在研究近地航天器测控设备分配策略时，针对“多 星冲突 “ 情况

评估指标

提出了任务满意度、 测控圈满意度、 冲突系数、 观测几何条件权系数和测 控设备综合系数等评估指标

资源调度

随着更多的卫星进入太空， 使地面站系统面临越来越严重的多星同时过站和多星同时任务请求的形势。地面站系统资源所能支待的卫星数目是有限的，当同时支持多颗卫星时，这些资源就需要进行合理的调度、 分配， 使各个卫星的需求都得到满足。

多星问题

一个比较典型的例子， 在轨运行的两颗卫星同时提出任务申请， 如果同时经过只有一部设备的地面站， 那么就只能为其中的一颗卫星服 务， 对另一颗卫星暂时不进行服务， 这两颗卫星之间就出现了争用资源的冲突。 因此， 多 星的任务冲突以及冲突的解决是地面站系统使用的核心问题。

地面站系统资源能力评估的指标体系及计算方法

地面测控资源调度效能评估指标体系

本文从梳理影响测控资源调度效能评估的各类指标要素除法，构建地面测控资源调度效能评估指标体系，为测控资源调度、管理和后续规划建设提供了决策依据，促进测控资源调度的不断优化。

评估指标要素

资源负荷率

描述了资源对卫星任务实际的测控时间占全部能支持任务时间的百分比，其分为单个地面测控资源负荷率和全网资源负荷率

单个地面测控资源负荷率

单个资源在指定时间内已承担任务时间与最大可承担任务时间的比值

全网资源负荷率

所有地面测控资源在规定时间内已承担任务时间与最大可承担任务时间的比值

资源均衡度

全网测控资源已承担任务时间的标准差，用于衡量全网资源间负荷的离散程度

且提供相关公式

资源最大可用率

描述了全网资源空闲且可分配任务时间占最大可承担任务时间的百分比，是测控资源弹性能力的评估指标，集中反映了全网资源的可用余量与最大能力之间的关系。

支持卫星冗余度

在已制定资源任务计划的基础上，全网资源还能支持多少颗卫星的测控需求。用于评估全网资源的剩余能力。

任务满足率

效能评估指标体系

评估指标是进行地面测控资源调度效能评估的前提。

  1. 对测控资源进行任务统筹规划，制定合理的测控任务计划。

  2. 对各方向用户需求进行响应，满足用户的用网需求

三个方面入手

  1. 资源统筹能力
  2. 资源服务能力
  3. 全网弹性能力

区分指标要素层级，构建分层、多维的评估指标体系。

为评估指标体系建立3个层次

  1. 效能层
  2. 能力层
  3. 指标层

层次分析法进行效能评估

分别对能力层的资源统筹能力、资源服务能力、全网弹性能力下属的指标层进行层次分析

再对效能层-能力层进行层次分析

最后根据权重再sync得到综合权重
