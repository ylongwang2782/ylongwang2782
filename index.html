home assistant安装与基础配置

发表于 2022-06-23 Waline： 阅读次数： 本文字数： 774 阅读时长 ≈ 1 分钟

群晖docker安装home assistant

背景

因为特殊原因（qiong）家里购买了很多平台的智能设备，所以需要一个整合平台。home assistant就非常合适。

安装

  1. 打开docker， 选择注册表，搜索home assistant ，选择第一个收藏量最高的，下载映像
  2. 下载后启动，点击高级设置，在高级设置中： 卷设置为

网络设置为与host相同。

  1. 点击启动。
  2. 输入群晖host地址后面端口为默认8123，即，hostip:8123

以上则安装完成。

插件安装

sonoff插件安装

sonoff是一个性价比比较高的产品系列，我选用了他家的三开智能开关，平均一个60元，非常的实惠。

在home assistant中接入 sonoff的设备则需要安装插件

  1. 在GitHub中搜索SonoffLAN，或者直接跳转链接https://github.com/AlexxIT/SonoffLAN ，在release 中下载最新的源代码压缩文件。
  2. 解压文件夹，将文件夹中sonoff整个文件夹复制粘贴到docker中以下地址
  3. 文件放置完成后，重启home assistant。
  4. 打开home assistant - 配置 - 设备与服务 - 添加集成 - 搜索sonoff - 填写账号密码，则可以将sonoff的设备同步到ha中。

hacs插件安装与配置

原本的ha UI比较朴素，整个看起来不够美丽。

添加home assistant custom store 插件，则可以方便的挑选已有的美化配置。

  1. 同样是在GitHub中搜索hacs（https://github.com/hacs/integration/releases）下载最新版。

  2. 安装流程同上sonoff

  3. 安装成功后，在左侧栏中可以查询到hacs 的界面

  4. 在其中则可以选择集成或者前端下载，其中集成是一些功能性插件；前端则包括lovelace 和 主题下载。（该下载可能需要群晖开启代理）

群晖怎么访问GitHub

发表于 2022-06-23 Waline： 阅读次数： 本文字数： 326 阅读时长 ≈ 1 分钟

背景

因为安装hacs需要GitHub认证，但是家里的群晖无法访问GitHub，所以需要采取特殊手段。

方法设计

  • 群晖安装clash（太复杂）
  • 群晖使用pc的clash

综上，选择第二种方法。

实现方法

clash拥有 allow lan的功能，即局域网使用功能

  1. 在clash中开启allow lan
  2. cmd 中 输入ipconfig 查询本机ipv4地址，或者其他方法都可以。
  3. 群晖 - 控制面板 - 网络 - 代理服务器 - 地址填写pc的ip地址；端口填写默认的clash 端口 7890（或者修改后的端口）
  4. 勾选通过代理服务器连接，同时pc的clash开启proxy

以上，群晖就可以正常访问GitHub了
