基础设置

代理服务器设置

主要是关于代理服务器的设置，因为TMDB等连接没有代理则不能生成。

关于代理服务器，最简单的方法是用clash开启allow lan，在代理服务器直接填pc的ip地址。

刮削设置

如果要调用软件的刮削功能，则必须填写TMDB api

获取方法：开启代理，注册TMDB，获得账号的api

在刮削设置中，该软件的运行原理是硬链接的同时进行刮削，但刮削功能较弱。

刮削效果仅能分门别类电影，但不能刮削海报等图片。

但重命名和分类的效率比较高，使用方法也简单。

刮削的文件设置

因为硬链接仅支持同一个共享文件夹设置，所以在设置的时候，在一级目录设置电影，电视剧，纪录片等文件夹，并且同时设置一个LINKS文件夹，然后在二级文件夹LINKS中，对应的设置对应一级文件夹的链接文件夹，电影、电视剧、纪录片等。

在目录同步中设置对应的刮削地址即可。

索引器Jackett设置

安装方法：在docker中下载镜像安装即可

注意不要拉取latest，应该手动选择GitHub上对应的最新版本，因为自动拉去的latest版本是超旧的版本

相关设置：登录到Jackett的web管理界面，首先是添加indexer，也就是站点，bt和pt沾点都可以添加

pt站点设置：add indexer然后搜索对应的站点，点击扳手配置即可。

值得注意的是，馒头因为设置的cloudflare，所以在添加该站点的时候需要添加cloudsolverr api以跳过cloudflare检查。

我在设置cloudsolverr的途中遇到了很多麻烦

flaresolverr闪退

因为flaresolverr在启动时会检测网络环境

需要添加环境变量代理

http_proxy:http://192.168.0.228:7890

all_proxy:http://192.168.0.228:7890

没有闪退，仍旧是无法正常运行

自动拉取的版本过低

需要手动拉取最新的版本

仍然无法跳过馒头检验

issue上面针对该问题，说的是版本问题，在2.2.6的flaresolver solve this question
